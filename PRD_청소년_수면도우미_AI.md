# PRD · 잠코치 (SleepCoach) — 청소년 수면·스마트폰 습관 AI 도우미

> **한 줄 요약** — 청소년 절반이 자기가 잠이 부족한 줄도 모른다. 그걸 데이터로 비춰주고, 습관을 바꾸게 돕는 AI 도우미.

## 1. 배경 · 문제 정의 (당위성)

한국 청소년의 수면 부족과 스마트폰 과의존은 통계로 입증된 사회 문제다.

- **수면 부족, OECD 최하위** — 우리나라 청소년 주중 평균 수면 시간은 약 5.8시간으로 OECD 국가 중 가장 적다. 권장 시간은 8~10시간. 중학생의 약 52%, 고등학생의 약 90%가 7시간 미만으로 잔다.
- **정작 본인은 모른다** — 잠이 부족한 학생 중 약 35%만 스스로 부족하다고 인식한다. → *객관적으로 보여주는 도구가 필요하다는 직접적 근거.*
- **스마트폰 과의존, 청소년만 증가** — 전체 과의존 위험군은 5년 연속 감소했지만, 청소년(10~19세)은 43.0%로 오히려 늘었다. 숏폼·생성형 AI 확산이 원인으로 분석된다.
- **둘은 연결돼 있다** — 스마트폰 조절 실패는 정서 불안정을 높이고, 반대로 사용 습관이 안정되면 수면의 질도 좋아진다. → *폰 습관을 잡으면 수면이 나아지는 선순환.*

### 왜 지금 풀어야 하나
청소년기 수면 부족은 성인이 된 뒤의 학습·기억력까지 떨어뜨린다(해마 신경 재생 억제). 단순한 피로 문제가 아니다. 학업량·등교시간 같은 구조적 원인은 학생이 못 바꾸지만, **스마트폰 사용 습관과 수면 루틴은 스스로 통제 가능**하다. AI가 들어갈 자리는 바로 여기 — 본인도 모르는 패턴을 데이터로 비춰주고, 작은 행동 변화를 돕는 것.

---

## 2. 목표

### 제품 목표
사용자가 자기 수면·스마트폰 사용 패턴을 객관적으로 인식하고, 매일 작은 습관 하나를 바꾸도록 돕는다.

### 성공 기준 (포트폴리오 기준)
- 사용자가 3일 이상 연속 기록을 이어간다 (지속성)
- AI 분석이 "내가 몰랐던 패턴"을 한 가지 이상 짚어준다 (인사이트)
- 추천 미션이 실제로 해볼 만하다고 느낀다 (실행 가능성)

---

## 3. 타깃 사용자

- **주 사용자** — 스마트폰을 늦게까지 보다 수면이 부족한 중·고등학생
- **특징** — 입력이 조금만 귀찮아도 안 쓴다 / 잔소리를 싫어한다 / 자기 데이터를 보는 건 흥미로워한다
- **핵심 니즈** — "내가 진짜 잠이 부족한 건지, 폰을 얼마나 보는 건지" 객관적으로 알고 싶다

---

## 4. 사용자 시나리오

1. 자기 전, 오늘 잔 시간·폰 사용 시간·컨디션을 30초 만에 기록한다.
2. 며칠 쌓이면, AI가 "자기 전 폰 1시간 넘긴 날은 다음 날 컨디션이 낮았어요" 같은 패턴을 알려준다.
3. 오늘의 미션을 하나 받는다 — "11시 전에 충전기에 폰 꽂아두기".
4. 일주일 뒤, AI가 한 주를 요약해준다 — 잘한 점, 아쉬운 점.

## 5. 기능 요구사항

### 5-1. MVP (1순위 · 먼저 완성)

| # | 기능 | 설명 | AI 역할 |
|---|---|---|---|
| F1 | 데일리 기록 | 수면 시간, 자기 전 폰 사용 시간, 컨디션(1~5) 입력 (하루 30초) | — |
| F2 | 패턴 분석 | 일주일치 기록에서 의미 있는 패턴 1~2개 도출 | ◎ AI 핵심 |
| F3 | 오늘의 미션 | 분석에 맞춰 실행 가능한 미션 1개 추천 | ◎ AI |

> **MVP 정의** — "기록 → 일주일 패턴 분석 → 미션 1개 추천" 이 한 줄이 작동하면 MVP 완성.

#### 입력 항목 상세 (F1)

| 입력 | 형태 | 예시 |
|---|---|---|
| 어제 잔 시각 | 시간 선택 | 01:30 |
| 오늘 일어난 시각 | 시간 선택 | 07:00 |
| 자기 전 폰 사용 시간 | 슬라이더(분) | 90분 |
| 오늘 컨디션 | 1~5점 버튼 | 2점 |
| 낮에 졸렸나요 | 예 / 아니오 | 예 |

→ 수면 시간은 잔 시각·일어난 시각으로 **자동 계산**.

#### 분석 규칙 (F2) — AI가 짚어주는 패턴 예시

- 주중 평균 수면 < 권장(8시간) → "주중에 권장보다 N시간 부족해요"
- 자기 전 폰 사용 많은 날 vs 적은 날의 컨디션 차이 비교 → "폰 오래 본 날 컨디션이 평균 N점 낮았어요"
- 취침 시각 표준편차 큼 → "자는 시간이 매일 들쭉날쭉해요"
- 낮 졸림 '예' 비율 높음 → "이번 주 절반 이상 낮에 졸렸어요"

#### 미션 추천 규칙 (F3) — 분석 결과 → 미션 매핑

| 발견된 문제 | 추천 미션 |
|---|---|
| 자기 전 폰 사용 과다 | "오늘은 11시 전에 폰 충전기에 꽂아두기" |
| 취침 시각 불규칙 | "오늘은 어제와 같은 시각에 누워보기" |
| 수면 시간 부족 | "오늘은 30분만 일찍 불 끄기" |
| 낮 졸림 잦음 | "점심 후 10분 가벼운 산책하기" |

### 5-2. 다음 단계 (2순위 · MVP 이후)

- 수면 점수 매기기 (수면시간·규칙성·폰사용 기반 100점 환산, 매일 점수로 동기 부여)
- 주간 리포트 (AI가 한 주를 요약: 잘한 점 / 아쉬운 점 / 다음 주 제안)
- 폰 사용 ↔ 수면 상관관계 그래프 시각화 *(← "35%만 인지" 문제를 정확히 때리는 기능)*
- 응원 메시지 톤 조정 (잔소리 X, 친구처럼)

### 5-3. 이번 범위 밖 (Out of Scope)
- 실제 수면 자동 측정(웨어러블 연동), 의료적 진단, 타인과의 비교/랭킹, 회원 가입·서버 저장, 알림 푸시

## 6. AI 설계 포인트

- **단순 평균 ❌ → 패턴 발견 ⭕** — "평균 6시간"이 아니라 "주중에 권장보다 2시간 부족", "취침 시간이 들쭉날쭉" 같은 통찰을 준다.
- **자연어 코멘트** — 규칙으로 찾은 패턴을 LLM이 사람 말투로 풀어 설명 (바이브 코딩 + LLM 활용).
- **역할 분담** — *AI 분석 로직(F2)·미션 매핑(F3) 설계는 승준이가 직접*, 화면(프론트)·데이터 연결(백엔드)은 바이브 코딩으로 빠르게 구현.

## 7. 기술 방향 (확정안)

| 영역 | 선택 | 이유 |
|---|---|---|
| 형태 | 웹앱 (반응형, 모바일 우선) | 설치 없이 폰에서 바로 사용 |
| 프론트엔드 | HTML / CSS / JavaScript (바이브 코딩) | 가볍고 빠르게 구현 |
| 데이터 저장 | 브라우저 로컬 저장(LocalStorage) | 서버 없이 MVP 완주 가능 |
| AI 코멘트 | LLM API 호출 (분석 결과 → 자연어 변환) | 똑똑한 도우미 느낌 |
| 분석 로직 | JavaScript 규칙 기반 + AI 코멘트 | 본인이 설계하는 핵심 영역 |

> 화면 구성: ① 입력 화면 → ② 분석 결과 화면 → ③ 오늘의 미션 카드. 3개 화면으로 시작.

## 8. 일정 (4주 계획)

| 시기 | 목표 | 산출물 |
|---|---|---|
| 1주차 | 명세서 확정 + 입력 화면(F1) | 작동하는 입력 폼 + 자동 수면시간 계산 |
| 2주차 | 패턴 분석 로직(F2) | 일주일 데이터로 패턴 1~2개 도출 |
| 3주차 | 미션 추천(F3) 연결 + AI 코멘트 | MVP 완성 (기록→분석→미션) |
| 4주차 | 2순위 기능 1개(수면 점수) + 노션 정리 | 데모 영상 + 포트폴리오 페이지 |
