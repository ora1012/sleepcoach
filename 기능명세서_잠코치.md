# 상세 기능 명세서 · 잠코치 (SleepCoach)

> 본 문서는 [PRD]의 기능을 개발 기준으로 상세화한 것이다. 화면 → 입력/출력 → 동작 → 데이터 → 예외 순으로 정의한다.

| 항목 | 내용 |
|---|---|
| 작성자 | 장승준 |
| 버전 | v0.1 |
| 대상 | MVP (F1 기록 / F2 분석 / F3 미션) |
| 플랫폼 | 웹앱 (모바일 우선 반응형) |

## 0. 화면 구조 (Sitemap)

```
[홈/입력 화면]  ──기록 저장──▶  [분석 결과 화면]  ──▶  [오늘의 미션 카드]
      ▲                                                      │
      └──────────────────  "다시 기록하기"  ◀────────────────┘
```

- 화면 3개: ① 입력 → ② 분석 결과 → ③ 미션
- 하단 고정 탭: [기록] [내 데이터] (2탭으로 시작)

---

## F1. 데일리 기록

### 화면
홈 화면 = 입력 폼. 오늘 날짜 자동 표시. 이미 오늘 기록이 있으면 "수정" 모드로 전환.

### 입력 항목

| ID | 항목 | UI 형태 | 값 범위 | 필수 | 기본값 |
|---|---|---|---|---|---|
| in1 | 어제 잔 시각 | 시간 선택기 | 00:00~23:59 | ✅ | 23:00 |
| in2 | 오늘 일어난 시각 | 시간 선택기 | 00:00~23:59 | ✅ | 07:00 |
| in3 | 자기 전 폰 사용 시간 | 슬라이더 | 0~240분 (10분 단위) | ✅ | 60 |
| in4 | 오늘 컨디션 | 1~5 버튼 | 1~5 | ✅ | 없음 |
| in5 | 낮에 졸렸나요 | 토글 | 예/아니오 | ✅ | 아니오 |

### 동작
1. 사용자가 5개 항목 입력 → [저장] 버튼 활성화 (in4 미선택 시 비활성).
2. 저장 시 **수면 시간 자동 계산**: `wake_time - sleep_time` (자정 넘김 처리).
   - 예: 잔 시각 01:30, 일어난 시각 07:00 → 5.5시간.
3. 저장 후 → 분석 결과 화면으로 이동.

### 데이터 모델 (기록 1건)

```json
{
  "date": "2026-06-14",
  "sleepTime": "01:30",
  "wakeTime": "07:00",
  "sleepHours": 5.5,
  "phoneMinutes": 90,
  "condition": 2,
  "daySleepy": true
}
```

저장 위치: 브라우저 LocalStorage, 키 `sleepcoach_records` (배열).

### 예외 처리
- 잔 시각 = 일어난 시각 → "시간을 다시 확인해주세요" 안내.
- 계산된 수면 시간 > 14시간 → "정말 N시간 주무셨나요?" 재확인.
- 같은 날짜 재입력 → 기존 기록 덮어쓰기(수정).

## F2. 패턴 분석 (AI 핵심)

### 화면
분석 결과 화면. 상단에 "이번 주 한눈에", 아래에 발견된 패턴 카드 1~2개 + AI 코멘트.

### 동작 조건
- 기록 **3건 이상** 쌓이면 분석 시작 (그 전엔 "N일만 더 기록하면 분석을 시작해요" 안내).
- 최근 7일치 데이터 기준.

### 분석 규칙 (규칙 기반 → 우선순위 순으로 1~2개 노출)

| 우선순위 | 조건 | 출력 메시지 (예시) |
|---|---|---|
| 1 | 주중 평균 수면 < 8시간 | "주중에 권장(8시간)보다 평균 {N}시간 부족해요" |
| 2 | 폰 많이 본 날 컨디션 - 적게 본 날 컨디션 ≥ 1점 | "폰을 오래 본 날 컨디션이 평균 {N}점 낮았어요" |
| 3 | 취침 시각 표준편차 ≥ 60분 | "자는 시간이 매일 들쭉날쭉해요" |
| 4 | 낮 졸림 '예' 비율 ≥ 50% | "이번 주 절반 이상 낮에 졸렸어요" |

> 규칙은 JavaScript로 계산. **이 규칙 설계가 승준이가 직접 담당하는 핵심 영역.**

### AI 코멘트 (LLM)
- 위에서 찾은 패턴 데이터를 LLM에 전달 → 친구 같은 말투의 한 문단 코멘트 생성.
- **프롬프트 예시**:
  > "다음은 한 청소년의 일주일 수면 데이터 분석 결과야: {분석결과}. 잔소리하지 말고, 친구처럼 따뜻하게 2~3문장으로 코멘트해줘. 마지막에 작은 응원 한마디."

### 출력 데이터 (분석 결과)

```json
{
  "weekAvgSleep": 6.2,
  "patterns": [
    { "type": "sleep_short", "message": "주중에 권장보다 평균 1.8시간 부족해요" },
    { "type": "phone_condition", "message": "폰을 오래 본 날 컨디션이 평균 1.5점 낮았어요" }
  ],
  "aiComment": "이번 주 좀 무리했네! ... 오늘은 조금만 일찍 누워보자 :)"
}
```

### 예외 처리
- 기록 3건 미만 → 분석 대신 안내 문구.
- 모든 규칙 미해당 → "이번 주는 꽤 잘 잤어요! 지금 패턴을 유지해봐요" (긍정 메시지).
- LLM 호출 실패 → AI 코멘트 생략하고 규칙 메시지만 표시 (서비스 중단 X).

## F3. 오늘의 미션

### 화면
미션 카드 1개. 큰 글씨로 미션 + [완료] 체크 버튼.

### 동작
1. F2에서 나온 **1순위 패턴**에 맞춰 미션 1개 매핑.
2. 미션 매핑 테이블에서 해당 미션 노출.
3. 사용자가 [완료] 누르면 → 오늘 미션 완료 기록, "잘했어요!" 피드백.

### 미션 매핑 테이블

| 패턴 type | 추천 미션 |
|---|---|
| sleep_short | "오늘은 30분만 일찍 불 끄기" |
| phone_condition | "오늘은 11시 전에 폰 충전기에 꽂아두기" |
| sleep_irregular | "오늘은 어제와 같은 시각에 누워보기" |
| day_sleepy | "점심 후 10분 가벼운 산책하기" |
| (패턴 없음) | "오늘도 좋은 잠 자기! 자기 전 스트레칭 한 번" |

### 출력 데이터

```json
{
  "date": "2026-06-14",
  "missionType": "phone_condition",
  "missionText": "오늘은 11시 전에 폰 충전기에 꽂아두기",
  "completed": false
}
```

저장: LocalStorage 키 `sleepcoach_missions`.

### 예외 처리
- 패턴이 없으면 기본 미션 노출.
- 하루 1미션 고정 (여러 패턴이어도 1순위 1개만).

## 5-2 이후 기능 (2순위 · 참고용 요약)

| 기능 | 핵심 동작 | 추가 데이터 |
|---|---|---|
| 수면 점수 | 수면시간(40)+규칙성(30)+폰사용(30) → 100점 환산 | `score` 필드 |
| 주간 리포트 | 7일 모이면 잘한 점/아쉬운 점 AI 요약 | 주차별 집계 |
| 상관관계 그래프 | 폰 사용(x) ↔ 컨디션(y) 산점도 | 기존 데이터 활용 |
| 미션 완료율 | 완료한 미션 / 전체 미션 % | `completed` 집계 |

## 데이터 저장 구조 (전체)

| 키 | 내용 | 형태 |
|---|---|---|
| `sleepcoach_records` | 일별 기록 전체 | 배열 |
| `sleepcoach_missions` | 일별 미션 + 완료 여부 | 배열 |

> 서버 없음. 전부 브라우저 로컬에 저장 (MVP 범위). 데이터 초기화 버튼 1개 제공.

---

## 비기능 요구사항

- **속도** — 입력~저장 1초 이내, 분석 2초 이내.
- **사용성** — 입력 완료까지 30초 이내 (탭 최소화).
- **반응형** — 모바일 세로 화면 기준 우선 디자인.
- **개인정보** — 이름·연락처 등 식별 정보 수집 안 함. 로컬에만 저장.
- **접근성** — 버튼 크게, 글씨 충분히 크게 (청소년 폰 사용 기준).
