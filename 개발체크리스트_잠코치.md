# 개발 체크리스트 · 잠코치 (SleepCoach)

> 상세 기능 명세서를 개발 작업 단위로 쪼갠 체크리스트. 위에서부터 순서대로 진행하면 MVP가 완성된다.
> 진행하면서 `[ ]` → `[x]` 로 바꾸고, 막힌 곳은 맨 아래 "막힌 기록"에 적기.

| 항목 | 내용 |
|---|---|
| 개발자 | 장승준 |
| 목표 | MVP 완성 (기록 → 분석 → 미션) |
| 방식 | 바이브 코딩 (HTML/CSS/JS + LLM API) |

---

## STEP 0. 준비

- [x] 깃허브 저장소 생성 (`sleepcoach`)
- [x] 작업 폴더 만들기 (index.html, style.css, script.js)
- [x] PRD·기능명세서 노션에 정리해두기
- [x] 화면 3개 구조 머릿속/종이에 스케치 (입력 → 분석 → 미션)

---

## STEP 1. F1 — 데일리 기록 (입력)

### 1-1. 입력 화면 만들기
- [x] 오늘 날짜 자동 표시
- [x] 어제 잔 시각 입력 (시간 선택기, 기본 23:00)
- [x] 오늘 일어난 시각 입력 (시간 선택기, 기본 07:00)
- [x] 자기 전 폰 사용 시간 (슬라이더 0~240분, 10분 단위, 기본 60)
- [x] 오늘 컨디션 (1~5 버튼)
- [x] 낮에 졸렸나요 (예/아니오 토글)
- [x] [저장] 버튼 (컨디션 미선택 시 비활성화)

### 1-2. 수면 시간 자동 계산
- [x] 잔 시각·일어난 시각으로 수면 시간 계산
- [x] 자정 넘김 처리 (예: 01:30 취침 → 07:00 기상 = 5.5시간)

### 1-3. 데이터 저장
- [x] LocalStorage `sleepcoach_records`에 배열로 저장
- [x] 저장 데이터 형태 맞추기 (date, sleepTime, wakeTime, sleepHours, phoneMinutes, condition, daySleepy)
- [x] 저장 후 분석 결과 화면으로 이동

### 1-4. 예외 처리
- [x] 잔 시각 = 일어난 시각 → 안내 메시지
- [x] 수면 시간 14시간 초과 → 재확인 메시지
- [x] 같은 날짜 재입력 → 기존 기록 덮어쓰기(수정 모드)

✅ **STEP 1 완료 기준**: 입력하고 저장하면 데이터가 쌓이고, 수면 시간이 자동 계산된다.

---

## STEP 2. F2 — 패턴 분석 (AI 핵심)

### 2-1. 분석 시작 조건
- [x] 기록 3건 미만이면 "N일만 더 기록하면 분석 시작" 안내
- [x] 최근 7일치 데이터 불러오기

### 2-2. 분석 규칙 구현 (★ 본인이 직접 설계하는 핵심)
- [x] 규칙1: 주중 평균 수면 < 8시간 → 부족 시간 계산
- [x] 규칙2: 폰 많이 본 날 vs 적게 본 날 컨디션 차이 ≥ 1점
- [x] 규칙3: 취침 시각 표준편차 ≥ 60분 (들쭉날쭉)
- [x] 규칙4: 낮 졸림 '예' 비율 ≥ 50%
- [x] 우선순위 순으로 패턴 1~2개만 노출

### 2-3. AI 코멘트 (LLM)
- [x] 분석 결과를 LLM에 전달하는 프롬프트 작성 (친구 말투, 2~3문장)
- [x] LLM API 호출 연결
- [x] 받은 코멘트를 화면에 표시

### 2-4. 분석 결과 화면
- [x] "이번 주 한눈에" 요약 (평균 수면 등)
- [x] 발견된 패턴 카드 1~2개
- [x] AI 코멘트 한 문단

### 2-5. 예외 처리
- [x] 모든 규칙 미해당 → 긍정 메시지 ("이번 주는 잘 잤어요!")
- [x] LLM 호출 실패 → 코멘트 생략, 규칙 메시지만 표시 (중단 X)

✅ **STEP 2 완료 기준**: 기록이 쌓이면 패턴을 짚어주고, AI가 코멘트를 달아준다.

---

## STEP 3. F3 — 오늘의 미션

### 3-1. 미션 매핑
- [x] 1순위 패턴 → 미션 1개 매핑 테이블 구현
- [x] sleep_short → "30분만 일찍 불 끄기"
- [x] phone_condition → "11시 전에 폰 충전기에 꽂아두기"
- [x] sleep_irregular → "어제와 같은 시각에 누워보기"
- [x] day_sleepy → "점심 후 10분 산책하기"
- [x] 패턴 없음 → 기본 미션

### 3-2. 미션 카드 화면
- [x] 미션 1개 큰 글씨로 표시
- [x] [완료] 체크 버튼
- [x] 완료 시 "잘했어요!" 피드백

### 3-3. 미션 저장
- [x] LocalStorage `sleepcoach_missions`에 저장 (date, missionType, missionText, completed)
- [x] 하루 1미션 고정

✅ **STEP 3 완료 기준**: 분석에 맞는 미션이 나오고, 완료 체크가 된다. → **MVP 완성!**

---

## STEP 4. 마무리 (MVP 점검)

- [x] "다시 기록하기"로 입력 화면 돌아가기
- [x] 데이터 초기화 버튼 1개
- [x] 모바일 세로 화면에서 잘 보이는지 확인
- [x] 가상 데이터 7일치로 전체 흐름 테스트
- [x] 데모 영상 찍기 (1~2분)
- [x] 노션 포트폴리오에 스크린샷·설명 정리

---

## STEP 5. 2순위 기능 (여유 되면 1개만)

- [ ] 수면 점수 (수면시간 40 + 규칙성 30 + 폰사용 30 = 100점)
- [ ] 주간 리포트 (7일 모이면 AI 요약)
- [ ] 폰 사용 ↔ 컨디션 상관관계 그래프
- [ ] 미션 완료율 표시

> ⚠️ MVP(STEP 1~4) 끝나기 전엔 절대 손대지 않기. 욕심이 미완성을 부른다.
